一個純前端、免後端的互動式測驗系統，適合小學三年級科學評量使用。

## 📚 功能總覽

### 學生測驗

- 選擇班級（甲～己）、輸入姓名、學號（1-40）、密碼登入
- 題目隨機排序
- 四種題型：**圖片選擇題**、**文字選擇題**、**簡答題**、**填充題**
- 36 分鐘倒數計時
- 答題狀態導航（灰＝未做、綠＝已做、藍＝目前）
- 儲存答案（按鈕閃爍回饋）
- 查看答案（需密碼）
- 匯出 XLSX（含批改，三 Sheet 格式）
- 電郵提交答案

### 管理後台（題庫管理）

- **題庫管理**：新增/編輯/刪除四種題型題目
- **出題設定**：自動出題（依題型設定數量隨機抽題）或手動選題
- **匯出 / 匯入 XLSX**：題庫可存成 Excel 檔編輯後再匯入
- **重置題庫**：需輸入管理員密碼確認
- **從伺服器同步**：抓取網站目錄下的 `questions.xlsx`

### 管理後台（GitHub 發佈）

- 設定 GitHub Token + 倉儲名稱
- **一鍵發佈題庫**到 GitHub 倉儲，GitHub Pages 自動更新
- **從 GitHub 下載**最新題庫
- 所有老師共用同一份線上題庫

### 管理後台（☁️ GAS 雲端同步）

- 設定 Google Apps Script API 網址與密碼
- **測試連線**：驗證 GAS 部署是否正常
- **上傳 XLSX 到雲端**：將本地題庫打包上傳到 Google Drive
- **從雲端同步題庫**：從 Google Drive 下載最新 XLSX 並覆蓋本地題庫
- 所有操作均有即時紀錄顯示

### GitHub vs GAS 雲端對比

| 特性 | GitHub 發佈 | GAS 雲端同步 |
|------|-------------|--------------|
| **用途** | 將題庫發佈到 GitHub Pages 網站 | 將題庫備份到 Google Drive |
| **需要** | GitHub Personal Access Token | Google Apps Script 部署 |
| **學生存取** | ✅ 可透過網站自動載入 | ❌ 僅供老師備份/同步 |
| **協作編輯** | ❌ 需手動更新 | ✅ 可搭配 Google Sheets |
| **適合場景** | 正式題庫發佈 | 題庫備份與多裝置同步 |

> **建議工作流程**：編輯題庫 → 上傳到 GAS 雲端備份 → 發佈到 GitHub 讓學生使用

### 管理後台（考試結果）

- 匯入學生 XLSX 答案檔
- 自動合併全班成績
- 匯出全班總表（含正確率統計）

## 📋 XLSX 題庫格式

匯出的 `questions.xlsx` 包含以下欄位，可直接用 Excel 編輯：

| 欄位 | 說明 |
|------|------|
| ID | 題號 |
| 題型 | `image-choice` / `text-choice` / `short-answer` / `fill-blank` |
| 標題 | 題目標題 |
| 題目內容 | 題目文字（已去除 HTML） |
| 選項A~D | 選擇題選項文字 |
| 正確答案 | 選擇題填正確選項字母，填充題填答案文字 |
| 多組答案 | 填充題多組答案用 `|` 分隔（如 `液態\|液體`） |
| 圖片網址 | 圖片選擇題的圖片 URL |

編輯後存檔，放到網站目錄或透過後台匯入即可更新題庫。

## ☁️ GAS 雲端部署（推薦）

Google Apps Script 提供免費的雲端題庫管理，支援 Drive 檔案儲存與 Sheets 協作。

### 部署步驟

1. **建立 Google Sheet**
   - 到 [Google Sheets](https://sheets.google.com) 建立新試算表
   - 點選「延伸應用程式」→「Apps Script」

2. **貼上 Code.gs**
   - 將 `Code.gs` 內容貼到 Apps Script 編輯器
   - 儲存專案（Ctrl+S）

3. **部署為網頁應用程式**
   - 點選右上角「部署」→「新增部署」
   - 類型選「網頁應用程式」
   - 說明：「科學題庫 API」
   - 執行身分：**我**
   - 存取權：**任何人**（重要！）
   - 點選「部署」→ 授權 → 允許

4. **取得 API 網址**
   - 複製部署後的「網頁應用程式網址」
   - 格式：`https://script.google.com/macros/s/XXXXX/exec`

5. **設定前端**
   - 開啟管理後台 → 切換到「☁️ GAS 雲端」分頁
   - 貼上 API 網址
   - 密碼預設為 `admin`
   - 按「🔍 測試連線」驗證

### GAS 功能說明

| 功能 | 說明 |
|------|------|
|  上傳題庫到雲端 | 將本地題庫打包成 XLSX 上傳到 Google Drive |
| 🔄 從雲端同步題庫 | 從 Drive 下載最新 XLSX 並覆蓋本地題庫 |

### ❓ 常見問題

**Q: 出現 "FAIL TO FETCH" 錯誤？**
- 確認網址以 `/exec` 結尾（不是 `/edit`）
- 確認部署存取權設為「任何人」
- 確認已授權腳本執行（首次部署需手動執行一次）

**Q: GitHub 上傳失敗？**
- 確認 Personal Access Token 有效且有 `repo` 權限
- 確認倉儲名稱格式正確（如 `username/repo-name`）
- GitHub 上傳與 GAS 雲端是兩個獨立功能

## 🚀 部署方式

### GitHub Pages（推薦）

1. 將 `index.html` 上傳至 GitHub 倉儲
2. 將 `questions.xlsx` 放同目錄（可選，自動載入用）
3. 開啟 GitHub Pages
4. 老師在後台設定 GitHub Token + 倉儲名稱
5. 之後編輯題庫 → 按「發佈到 GitHub」→ 自動更新

### 本機離線測試

```bash
# 在 HTML 所在目錄開啟 HTTP 伺服器
python -m http.server 8080

# 開啟瀏覽器
http://localhost:8080
```

把 `questions.xlsx` 放同目錄，網頁會自動載入。

## 🛠 技術架構

- **純前端**：單一 HTML 檔案，無需後端伺服器
- **資料儲存**：題庫存在瀏覽器 `localStorage`
- **題庫同步**：透過 `questions.xlsx` 檔案 + GitHub API 共用題庫
- **Excel 處理**：使用 SheetJS 庫
- **樣式**：純 CSS，無框架依賴

## 📝 注意事項

- 學生密碼和管理員密碼可透過瀏覽器開發工具修改，建議定期更換

- GitHub Pages 更新約需 1-2 分鐘
